# PG Essays — Para uso en coaching

## Cómo agregar ensayos

Crea archivos markdown en este folder con el contenido de los ensayos de Paul Graham que quieras que el coach use como referencia.

Formato sugerido:

```
filename: do-things-that-dont-scale.md

---
title: Do Things That Don't Scale
author: Paul Graham
year: 2013
url: http://paulgraham.com/ds.html
---

[Texto completo del ensayo]
```

---

## Lista priorizada (sugerida)

Empezar con estos 5-7 ensayos. **No subir todo a la vez.**

1. **Do Things That Don't Scale** (2013) — http://paulgraham.com/ds.html
2. **Startup = Growth** (2012) — http://paulgraham.com/growth.html
3. **How to Get Startup Ideas** (2012) — http://paulgraham.com/startupideas.html
4. **The Hardest Lessons for Startups to Learn** (2006) — http://paulgraham.com/startuplessons.html
5. **Default Alive or Default Dead?** (2015) — http://paulgraham.com/aord.html
6. **How Not to Die** (2007) — http://paulgraham.com/die.html
7. **Mean People Fail** (2014) — http://paulgraham.com/mean.html

## Cómo se usan en coaching

Cuando una conversación toque un tema cubierto por algún ensayo:

1. El coach (en modo founder-coaching) los lee
2. Cita específicamente cuando aplique
3. Aplica el framework al contexto de Small Plates

No necesitan formato perfecto — texto plano funciona. El valor es que el contenido esté disponible.
